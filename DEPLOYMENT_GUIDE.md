# 🚀 دليل النشر والرفع الآمن لـ Prime على GitHub

مرحباً بك! لقد قمنا بتهيئة وضبط كافة تفاصيل المشروع البرمجية لتتمكن من رفعه إلى مستودعك على **GitHub** وبنائه وتعبئته بنجاح تام وسلاسة مطلقة وبدون أي عوائق أو توقفات. 

يحتوي هذا الدليل على شرح شامل لكيفية الرفع، وكيفية التعامل مع الصور والامتدادات، وكيفية تهيئة خوادم البناء الآلي (GitHub Actions).

---

## 🎨 1. حل مشكلة الأيقونات والصور (Tauri Icons)
واحدة من أكثر المشاكل شيوعاً والتي تؤدي لفشل بناء تطبيقات **Tauri** بالكامل في خوادم GitHub هي وجود أيقونات تالفة، أو بأحجام غير دقيقة، أو بامتدادات وهمية (مثل إعادة تسمية ملف `.jpg` إلى `.png` يدوياً دون تحويل فعلي).

### 🛠️ ما قمنا بعمله لحل هذه المشكلة:
لقد قمنا بتشغيل الأداة الرسمية لتوليد الأيقونات من Tauri عبر الأمر التالي:
```bash
npx tauri icon src-tauri/icons/256x256.png
```
* **النتيجة:** تم مسح كافة الأيقونات القديمة وإعادة توليد ملفات جديدة كلياً وبشكل متوافق 100% مع أنظمة التشغيل المختلفة:
  * ملف **`icon.icns`** (لنظام macOS) - تم توليده بحجم وبنية بايتات سليمة.
  * ملف **`icon.ico`** (لنظام Windows) - تم توليده بدقة ألوان وتدرجات صحيحة.
  * كافة أيقونات الـ **PNG** المخصصة للهواتف والويب بالأبعاد المطلوبة بدقة بكسل متناهية.

## 🔒 2. تحصين ملفات سير العمل (Resilient GitHub Workflows)
افتراضياً، كانت ملفات سير العمل في المجلد `.github/workflows/` (`ci.yml` و `release.yml`) تتوقف وتفشل إذا لم تقم بتهيئة مفاتيح الأسرار (GitHub Secrets) الخاصة بالتوكن الخاص بك `PRIME_UPDATE_PAT`.

### 🛠️ التحديثات الذكية التي أجريناها:
لقد قمنا بتعديل ملفات الـ Workflows برمجياً وإضافة فلاتر ذكية (`if` conditions):
* **تخطي مرن:** إذا قمت برفع المشروع على مستودع جديد أو فرعي ولم تكن قد قمت بتهيئة المتغيرات السرية بعد، **فلن يفشل البناء!** سيقوم النظام آلياً بتخطي خطوات المزامنة وحزم التحديثات والتركيز على نجاح بناء واختبار الكود البرمجي ليكون باللون الأخضر ✅ دائماً.

## 📋 3. دليل خطوات الرفع الفعلي إلى GitHub (Step-by-Step Push Guide)

اتبع هذه الخطوات البسيطة لرفع كافة التعديلات الجديدة دفعة واحدة وبأمان:

### 1️⃣ الخطوة الأولى: التحقق من الملفات المعدلة
افتح الطرفية (Terminal) في مجلد المشروع الرئيسي وشغّل الأمر التالي لمعاينة الملفات الجاهزة للرفع:
```bash
git status
```

### 2️⃣ الخطوة الثانية: إضافة الملفات إلى التخزين المؤقت (Stage Files)
شغّل الأمر التالي لإضافة التعديلات الجديدة وأيقونات المتصفح المحسنة إلى Git:
```bash
git add .
```

### 3️⃣ الخطوة الثالثة: تسجيل التعديلات (Commit)
قم بكتابة وصف معبر واحترافي للتحديثات الكبيرة التي قمنا بها:
```bash
git commit -m "feat: stabilize launcher loop, implement direct browser mode, add DOM OCR extractor and regenerate valid build icons"
```

### 4️⃣ الخطوة الرابعة: الرفع النهائي إلى المستودع (Push)
ارفع التعديلات إلى الفرع الرئيسي (master أو main) حسب التسمية لديك:
```bash
git push origin master
```

## ⚙️ 4. تهيئة أسرار المستودع على GitHub (Configuring GitHub Secrets)

لكي يقوم خادم البناء التلقائي ببناء نسخ كاملة وتوقيعها ورفعها للتحميل، يوصى بتهيئة المتغيرات التالية في إعدادات مستودعك على GitHub بالذهاب إلى:
**Settings ➡️ Secrets and variables ➡️ Actions ➡️ New repository secret**

| اسم المتغير (Secret Name) | الوصف والأهمية | طريقة الحصول عليه |
| :--- | :--- | :--- |
| `PRIME_UPDATE_PAT` | رمز الوصول الشخصي لدفع التحديثات البرمجية تلقائياً لمستودع التحديث الفرعي. | يتم توليده من إعدادات حسابك على GitHub تحت Developer Settings -> Personal Access Tokens. |
| `TAURI_SIGNING_PRIVATE_KEY` | المفتاح الخاص لتوقيع ملفات التثبيت (Debian/AppImage/Exe) لمنع تحذيرات الأمان في أنظمة التشغيل. | يتم توليده عبر أمر Tauri CLI: `npx tauri signer generate`. |
| `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` | كلمة المرور المشفرة الخاصة بمفتاح التوقيع أعلاه. | كلمة المرور التي اخترتها عند توليد المفتاح الخاص بالتوقيع. |

## 📸 5. نصائح هامة للتعامل مع الصور والامتدادات (Image Guidelines)

إذا أردت إضافة صور جديدة (شاشات عرض، أيقونات، صور توضيحية) في المستقبل، اتبع هذه القواعد الذهبية لمنع أي مشاكل بناء:

1. **لا تقم أبداً بإعادة تسمية الامتداد يدوياً:**
   * إذا كان لديك ملف `image.jpg` وتريد استخدامه كـ `image.png`، **لا تقم فقط بتغيير الاسم يدوياً**؛ لأن ترميز الملف الداخلي سيظل JPEG وسيؤدي لفشل المترجمات.
   * **الحل الصحيح:** افتح الصورة في أي برنامج لتعديل الصور أو موقع تحويل وقم بعمل "تصدير" (Export) أو "حفظ باسم" بصيغة PNG الحقيقية.
2. **استخدام سطر الأوامر للتحويل الآمن (في نظام لينكس):**
   يمكنك استخدام أداة `convert` الشهيرة المدمجة لتحويل الصور بشكل حقيقي وسليم 100%:
   ```bash
   convert input.jpg output.png
   ```
3. **أحجام الملفات:**
   * يفضل دائماً ضغط الصور التوضيحية الخاصة بـ `README` أو التوثيق باستخدام أدوات مثل [TinyPNG](https://tinypng.com/) أو صيغ حديثة مثل `WebP` ليبقى حجم مستودعك صغيراً وخفيفاً وسريع الرفع والدفع على GitHub!
